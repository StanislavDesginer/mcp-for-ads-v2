# Чеклист для человека без техзнаний

Этот файл нужен для самого простого сценария: человек получил репозиторий и должен просто понять, что делать дальше.

## Вариант 1. Просто проверить, что сайт работает

1. Открой в браузере:
   [http://77.240.38.131](http://77.240.38.131)
2. Убедись, что страница открывается.
3. Нажми обновление данных.
4. Проверь:
   - нет ли ошибок загрузки
   - видны ли блоки Meta dashboard
   - открывается ли диагностика

Если сайт открылся и данные грузятся, базовая проверка пройдена.

## Вариант 2. Проверить проект локально через Codex

### Что тебе должны дать

Попроси у владельца проекта:
- папку или репозиторий проекта
- файл `.env`
- файл `ads_config.yaml`

Без этих двух файлов реальные запросы работать не будут.

### Что установить на компьютер

Нужно:
- Python 3.11
- Git
- Codex

### Как поднять проект

1. Открой папку проекта `mcp-for-ads`
2. Открой PowerShell в этой папке
3. Выполни:

```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e ".[dev,google,meta]"
```

4. Убедись, что в корне проекта лежат:
   - `.env`
   - `ads_config.yaml`

### Как проверить веб-интерфейс локально

В этой же папке выполни:

```powershell
.\.venv\Scripts\python.exe -m ad_mcp.web.server
```

Потом открой:
- [http://127.0.0.1:8765](http://127.0.0.1:8765)

### Как проверить через Codex

1. Открой проект в Codex
2. Настрой MCP по примеру из:
   [.codex/config.example.toml](.codex/config.example.toml)
3. Перезапусти Codex
4. Напиши в чате одну из команд:

```text
Use MCP server ads and show list_accounts for provider meta_ads.
```

```text
Use MCP server ads and call get_account_summary for provider meta_ads and account_id act_1746501262698286.
```

## Что делать, если не работает

Проверь по порядку:

1. Есть ли Python 3.11
2. Есть ли `.env`
3. Есть ли `ads_config.yaml`
4. Поставились ли зависимости
5. Открывается ли [http://127.0.0.1:8765/healthz](http://127.0.0.1:8765/healthz)
6. Нет ли ошибки авторизации Meta
